# 📱 Быстрая настройка устройств

Краткое руководство по настройке различных устройств для работы с go-pcap2socks.

## ⚙️ Общие настройки для всех устройств

| Параметр | Значение |
|----------|----------|
| **Диапазон IP** | `172.26.0.2` – `172.26.255.254` |
| **Маска подсети** | `255.255.0.0` |
| **Шлюз** | `172.26.0.1` |
| **DNS (основной)** | `8.8.8.8` |
| **DNS (доп.)** | `1.1.1.1` |
| **MTU** | `1486` (или меньше) |

> **Примечание:** Значения могут отличаться в зависимости от вашей конфигурации. Смотрите вывод программы при запуске.

---

## 🎮 PlayStation 4 / PlayStation 5

### Пошаговая инструкция

1. **Настройки** → **Сеть** → **Настройки**
2. **Настроить подключение к Интернету**
3. Выберите **Использовать кабель LAN** (рекомендуется) или Wi-Fi
4. **Пользовательская настройка**
5. Заполните параметры:

```
Настройки IP-адреса: Вручную
├─ IP-адрес:        172.26.0.100
├─ Маска подсети:   255.255.0.0
├─ Шлюз:            172.26.0.1
├─ Основной DNS:    8.8.8.8
└─ Доп. DNS:        1.1.1.1

Настройки MTU: Вручную
└─ MTU: 1486

Прокси-сервер: Не использовать
```

6. **OK** → **Тест подключения к интернету**

### Проверка

После подключения проверьте:
- ✅ IP-адрес получен из диапазона `172.26.x.x`
- ✅ Интернет работает
- ✅ Ping проходит до шлюза `172.26.0.1`

## 🎮 Nintendo Switch

### Пошаговая инструкция

1. **Системные настройки** → **Интернет** → **Настройки интернета**
2. Выберите вашу сеть → **Изменить настройки**
3. **Настройки IPv4**: Вручную

```
├─ IP-адрес:        172.26.0.100
├─ Маска подсети:   255.255.0.0
├─ Шлюз:            172.26.0.1
├─ Основной DNS:    8.8.8.8
└─ Вторичный DNS:   1.1.1.1

MTU: Вручную
└─ MTU: 1486
```

4. **Сохранить** → **Тест подключения**

2. **Дополнительные настройки**

├─ IP-адрес:        172.26.0.100

## 📱 iOS (iPhone / iPad)

### Настройка Wi-Fi

1. **Настройки** → **Wi-Fi**
2. Нажмите **ⓘ** рядом с вашей сетью
3. **Настройка IP**: Вручную

```
├─ IP-адрес:        172.26.0.100
├─ Маска подсети:   255.255.0.0
├─ Маршрутизатор:   172.26.0.1
└─ DNS:             8.8.8.8
```

4. **Сохранить**

---

## 📱 Android

### Настройка Wi-Fi

1. **Настройки** → **Сеть и интернет** → **Wi-Fi**
2. Длительное нажатие на сеть → **Изменить сеть** / **Настройки**
3. **Дополнительно** → **Настройки IP**: Статический

```
├─ IP-адрес:            172.26.0.100
├─ Шлюз:                172.26.0.1
├─ Длина префикса:      16 (для маски 255.255.0.0)
├─ DNS 1:               8.8.8.8
└─ DNS 2:               1.1.1.1
```

4. **Сохранить**

> **Примечание:** Интерфейс может отличаться в зависимости от производителя устройства.

## 🖥️ Windows (ПК/Ноутбук)

### Настройка сетевого адаптера

1. **Панель управления** → **Сеть и Интернет** → **Центр управления сетями**
2. **Изменение параметров адаптера**
3. ПКМ на вашем подключении → **Свойства**
4. **IP версии 4 (TCP/IPv4)** → **Свойства**
5. **Использовать следующий IP-адрес**:

```
├─ IP-адрес:        172.26.0.100
├─ Маска подсети:   255.255.0.0
├─ Основной шлюз:   172.26.0.1
├─ Предпочитаемый DNS: 8.8.8.8
└─ Альтернативный DNS: 1.1.1.1
```

6. **OK** → **Закрыть**

---

## 🍎 macOS

### Настройка сети

1. **Системные настройки** → **Сеть**
2. Выберите ваш интерфейс (Wi-Fi/Ethernet)
3. **Подробнее...** → **TCP/IP**
4. **Настройка IPv4**: Вручную

```
├─ IP-адрес:        172.26.0.100
├─ Маска подсети:   255.255.0.0
└─ Маршрутизатор:   172.26.0.1
```

5. **DNS**: Добавьте `8.8.8.8` и `1.1.1.1`
6. **OK** → **Применить**

## 🐧 Linux

### Через терминал (NetworkManager)

```bash
# Статический IP для интерфейса eth0
sudo nmcli connection modify "YourConnection" \
    ipv4.addresses "172.26.0.100/16" \
    ipv4.gateway "172.26.0.1" \
    ipv4.dns "8.8.8.8,1.1.1.1" \
    ipv4.method manual

# Перезапуск подключения
sudo nmcli connection up "YourConnection"
```

### Через GUI

1. **Настройки** → **Сеть**
2. Выберите подключение → **⚙️**
3. **IPv4**: Вручную
4. Добавьте:
   - Адрес: `172.26.0.100`
   - Маска: `255.255.0.0`
   - Шлюз: `172.26.0.1`
   - DNS: `8.8.8.8,1.1.1.1`
5. **Применить**

## 🔍 Диагностика

### Проверка подключения

```bash
# Проверка связи со шлюзом
ping 172.26.0.1

# Проверка DNS
nslookup google.com

# Проверка маршрута
traceroute 8.8.8.8  # Linux/macOS
tracert 8.8.8.8     # Windows
```

### Если не работает

1. ✅ Убедитесь, что go-pcap2socks запущен
2. ✅ Проверьте, что IP из правильного диапазона
3. ✅ Перезапустите сетевое подключение на устройстве
4. ✅ Проверьте брандмауэр Windows
5. ✅ Убедитесь, что AmneziaVPN работает

### Логи go-pcap2socks

```bash
# Запуск с подробным логированием
SLOG_LEVEL=debug ./go-pcap2socks.exe
```
