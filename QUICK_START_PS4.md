## 🎮 Что читается из DHCP запроса PS4

Теперь сервис читает **все DHCP опции**:

| Option | Название | Пример значения |
|--------|----------|-----------------|
| **12** | Host Name | Имя хоста (например, "PS4-123") |
| **53** | Message Type | Discover/Request |
| **55** | Parameter Request List | [1,3,6,15,28,...] |
| **60** | Vendor Class Identifier | "MSFT 5.0" или "PSPC" |
| **61** | Client Identifier | Уникальный ID клиента |

**PS4 сама решает, какие параметры использовать** из тех, что отправляет сервер в DHCP OFFER/ACK.

## ❓ Решение проблем

### Сервис не запускается
1. Убедитесь, что запущен **от имени администратора**
2. Проверьте, что установлен **Npcap** (https://npcap.com)
3. Проверьте логи в `app.log`

### PS4 не видит сеть
1. Проверьте, что кабель подключён
2. Проверьте ARP таблицу: `arp -a | findstr 192.168.137`
3. PS4 должна быть: `192.168.137.100  78-c8-81-4e-55-15`

### NAT тип 3 (строгий)
1. Включите UPnP на роутере
2. Или пробросьте порты вручную:
   - **3478-3480 TCP/UDP** → `192.168.137.100`
