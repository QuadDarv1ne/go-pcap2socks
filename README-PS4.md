# go-pcap2socks - Настройка для PS4

## 🚀 Быстрый старт

### 1. Подготовка

**Скачайте WinDivert:**
- Перейдите на https://reqrypt.org/download.html
- Скачайте последнюю версию
- Распакуйте `WinDivert.dll` в папку с программой

### 2. Подключение PS4

**Вариант A: Ethernet кабель (рекомендуется)**
```
[Компьютер] ----(Ethernet кабель)---- [PS4]
```

**Вариант B: Wi-Fi хотспот**
1. Windows: Параметры → Сеть и Интернет → Мобильный хотспот → Включить
2. PS4: Настройки → Сеть → Настройки соединения → Выберите Wi-Fi сеть хотспота

### 3. Настройка сети на PS4

1. **Настройки** → **Сеть** → **Настройки соединения**
2. Выберите ваше подключение (Ethernet или Wi-Fi)
3. **Настройки IP:** Вручную
4. Введите параметры:
   ```
   IP адрес:      192.168.100.100
   Маска подсети: 255.255.255.0
   Шлюз:          192.168.100.1
   DNS:           8.8.8.8
   ```

### 4. Запуск программы

**От имени администратора!**

- Дважды кликните на `start.bat`
- Или правой кнопкой на `go-pcap2socks.exe` → "Запуск от имени администратора"

### 5. Проверка

1. **На PS4:** Настройки → Сеть → Состояние соединения
2. Должно показать: **Подключено**
3. **Проверьте интернет:** "Проверить подключение к Интернету"

## 🔧 Конфигурация

### config.json

Основные настройки в `config.json`:

```json
{
    "pcap": {
        "network": "192.168.100.0/24",
        "localIP": "192.168.100.1",
        "localMAC": "xx:xx:xx:xx:xx:xx"
    },
    "dhcp": {
        "enabled": true,
        "poolStart": "192.168.100.100",
        "poolEnd": "192.168.100.200"
    },
    "routing": {
        "rules": [
            {"dstPort": "53", "outboundTag": "dns-out"},
            {"outboundTag": "direct"}
        ]
    }
}
```

## ❌ Решение проблем

### "Сетевой носитель отключен"

**Проблема:** Кабель Ethernet не подключен

**Решение:**
1. Подключите кабель от компьютера к PS4
2. Или включите Wi-Fi хотспот

### PS4 не получает IP

**Проблема:** DHCP сервер не работает

**Решение:**
1. Проверьте, что WinDivert.dll в папке с программой
2. Запустите от имени администратора
3. В логах должно быть: "WINDIVERT DHCP SERVER STARTED"

### Интернет не работает

**Проблема:** Трафик не маршрутизируется

**Решение:**
1. Проверьте config.json — должно быть правило `{"outboundTag": "direct"}`
2. Проверьте логи — должно быть "TCP dial success"

### "429 Too Many Requests"

**Проблема:** Google ограничивает проверки

**Решение:** Это нормально. Интернет работает, просто health check не проходит.
