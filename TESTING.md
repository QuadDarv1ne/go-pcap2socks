# Тестирование

## Быстрые тесты (локальная разработка)

Для быстрого запуска тестов **без** детектора гонок (не потребляет много ОЗУ):

### Windows
```bat
test.bat
```

### Linux/macOS
```bash
./test.sh
```

**Параметры:**
- `-short` — пропускает длительные тесты
- `-run '^(Test|Example)'` — только unit-тесты (исключает fuzz и бенчмарки)
- `-p 1` — ограничивает параллелизм (меньше потребление памяти)
- `-timeout=2m` — максимум 2 минуты на тест

**Время выполнения:** ~10-30 секунд  
**Потребление ОЗУ:** низкое

---

## Тесты с детектором гонок (CI/отладка)

Для запуска с детектором гонок (**высокое** потребление ОЗУ):

### Windows
```bat
test-race.bat
```

### Linux/macOS
```bash
./test-race.sh
```

**Параметры:**
- `-race` — включает детектор гонок
- `-timeout=5m` — максимум 5 минут на тест

**Время выполнения:** ~5-10 минут  
**Потребление ОЗУ:** высокое (в 5-10 раз больше обычного)

> ⚠️ **Предупреждение:** Эти тесты могут вызвать лаги системы. Используйте только в CI или при отладке гонок.

---

## Запуск отдельных тестов

### Конкретный пакет
```bash
go test -v ./proxy/...
```

### Конкретный тест
```bash
go test -v -run TestRouter ./proxy/...
```

### Только fuzz-тесты
```bash
go test -fuzz=Fuzz -fuzztime=10s ./transport/...
```

### Только бенчмарки
```bash
go test -bench=. -run=^$ ./proxy/...
```

---

## Почему тесты лагают?

| Причина | Влияние | Решение |
|---------|---------|---------|
| `-race` (детектор гонок) | 10× ОЗУ, 20× медленнее | Использовать `test.bat` для локальных тестов |
| Fuzz-тесты | Высокое CPU | Исключены из `test.bat` через `-run` |
| Бенчмарки с `RunParallel` | Высокое CPU/ОЗУ | Исключены из `test.bat` через `-short` |
| Параллелизм (`-p`) | Скачки ОЗУ | Ограничено до `-p 1` в `test.bat` |

---

## Рекомендации

1. **Локальная разработка:** Используйте `test.bat` / `test.sh`
2. **Перед коммитом:** Запустите `test-race.bat` / `test-race.sh` один раз
3. **CI/CD:** Автоматически запускает оба типа тестов
