# Руководство по запуску тестов

## ⚠️ Проблема с потреблением памяти

Тесты в проекте могут потреблять значительное количество ОЗУ из-за:
- **Бенчмарков с `RunParallel`** — создают много goroutines одновременно
- **Fuzz-тестов** — генерируют большие объёмы данных
- **Тестов с worker pools** — держат активные goroutines
- **Кэшей и пулов** — могут расти без ограничений

## 🚀 Быстрый запуск (рекомендуется)

Для повседневной разработки используйте оптимизированный скрипт:

```bash
# Windows
test.bat

# Linux/macOS
./test.sh
```

Этот скрипт:
- ✅ Устанавливает `GOMEMLIMIT=1GB` для защиты от переполнения
- ✅ Отключает race detector (экономит ~2x памяти)
- ✅ Пропускает fuzz-тесты и бенчмарки
- ✅ Ограничивает параллелизм (`-p 2 -parallel 2`)
- ✅ Использует `-short` для пропуска тяжёлых тестов

## 📋 Доступные команды

### Обычные тесты
```bash
# Быстрый прогон (без fuzz и бенчмарков)
test.bat

# Сvoими параметрами
test.bat 10m ./proxy/...  # timeout 10 мин, только proxy пакет
```

### Бенчмарки
```bash
# Windows (с ограничением памяти 2GB)
DISABLED_bench.bat

# Linux/macOS
DISABLED_bench.sh

# Свои параметры
DISABLED_bench.bat 5s ./bufpool/...  # benchtime 5s, только bufpool
```

### Полные тесты (не рекомендуется)
```bash
# Запуск ВСЕХ тестов включая fuzz и benchmarks
# ⚠️ Может потребовать 4-8GB ОЗУ
go test -timeout=30m ./...
```

### Отдельные пакеты
```bash
# Тестирование конкретного пакета
go test -short ./proxy/...

# С покрытием
go test -short -cover ./proxy/...

# С выводом логов
go test -short -v ./proxy/...
```

## 🔧 Настройки памяти

### GOMEMLIMIT

Устанавливает мягкий лимит памяти для Go runtime:

```bash
# Windows
set GOMEMLIMIT=1073741824  # 1GB
set GOMEMLIMIT=2147483648  # 2GB
set GOMEMLIMIT=4294967296  # 4GB

# Linux/macOS
export GOMEMLIMIT=1073741824
export GOMEMLIMIT=2147483648
```

**Рекомендации:**
- **1GB** — для быстрой разработки
- **2GB** — для полного тестирования
- **4GB+** — для CI/CD или финальной проверки

### CGO_ENABLED

Race detector увеличивает потребление памяти ~2x:

```bash
# Без race detector (быстрее, меньше памяти)
set CGO_ENABLED=0

# С race detector (медленнее, больше памяти)
set CGO_ENABLED=1
```

## 📊 Типичное потребление памяти

| Режим | Память | Время | Когда использовать |
|-------|--------|-------|-------------------|
| `test.bat` (быстрый) | 0.5-1GB | 1-3 мин | Повседневная разработка |
| `test.bat` (полный) | 1-2GB | 5-10 мин | Перед коммитом |
| `DISABLED_bench.bat` | 2-4GB | 5-15 мин | Оптимизация производительности |
| `go test ./...` | 4-8GB | 10-30 мин | CI/CD, финальная проверка |

## 🐛 Если тесты падают по памяти

### Симптомы
- `runtime: out of memory`
- `fatal error: runtime: out of memory`
- Система начинает сильно тормозить

### Решения

1. **Уменьшите GOMEMLIMIT:**
   ```bash
   set GOMEMLIMIT=536870912  # 512MB
   ```

2. **Запустите только нужный пакет:**
   ```bash
   go test -short ./proxy/...
   ```

3. **Отключите параллелизм:**
   ```bash
   go test -short -p 1 -parallel 1 ./...
   ```

4. **Исключите тяжёлые тесты:**
   ```bash
   go test -short -run "^(Test|Example)" ./...
   ```

## 🏷️ Исключение тяжёлых тестов

### Fuzz-тесты
Исключаются автоматически при использовании `-short` или `-run "^(Test|Example)"`

### Бенчмарки
Не запускаются без флага `-bench`:
```bash
# Без бенчмарков
go test -short ./...

# С бенчмарками
go test -bench=. -benchtime=1s ./...
```

### Долгие тесты
Помечаются как `t.Skip()` или используют `testing.Short()`:
```bash
# Пропустить долгие тесты
go test -short ./...
```

## 📁 Структура тестов

```
project/
├── *_test.go          # Обычные тесты (Test*)
├── *_bench_test.go    # Бенчмарки (Benchmark*)
├── *_fuzz_test.go     # Fuzz-тесты (Fuzz*)
└── test.bat           # Скрипт быстрого запуска
```

## ✅ Best Practices

1. **Всегда используйте `defer` или `t.Cleanup()`** для освобождения ресурсов
2. **Ограничивайте размер пулов и кэшей** в тестах
3. **Вызывайте `runtime.GC()`** в `t.Cleanup()` для тяжёлых тестов
4. **Избегайте `time.Sleep()`** в тестах, используйте каналы и таймауты
5. **Помечайте долгие тесты** через `testing.Short()`

## 🔍 Отладка проблем с памятью

### Профилирование
```bash
# Запуск с профилем памяти
go test -memprofile=mem.out -short ./...

# Анализ профиля
go tool pprof mem.out

# Топ функций по памяти
go tool pprof -top mem.out
```

### Логирование
```bash
# Включить логирование GC
GODEBUG=gctrace=1 go test -short ./...

# Вывод статистики памяти
GODEBUG=memprofilerate=1 go test -short ./...
```
