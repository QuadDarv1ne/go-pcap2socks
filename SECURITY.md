# Политика безопасности

## Поддерживаемые версии

Мы рекомендуем всегда использовать последнюю стабильную версию go-pcap2socks.

| Версия | Поддерживается | Примечания |
|--------|---------------|------------|
| 3.18.x | ✅ Да | Текущая стабильная |
| 3.17.x | ⚠️ Частично | Только критические исправления |
| < 3.17 | ❌ Нет | Не поддерживается |

## Сообщение об уязвимостях

Мы серьёзно относимся к безопасности go-pcap2socks и благодарим за ответственное сообщение об уязвимостях.

### Как сообщить

**Пожалуйста, НЕ создавайте публичные issues для уязвимостей безопасности!**

Вместо этого отправьте отчёт на электронную почту:
- **Email**: [quaddarv1ne@example.com](mailto:quaddarv1ne@example.com)
- **GitHub**: Используйте [Private Vulnerability Reporting](https://github.com/QuadDarv1ne/go-pcap2socks/security/advisories/new)

### Что включать в отчёт

Для максимально быстрого решения проблемы, пожалуйста, предоставьте:

1. **Описание уязвимости**
   - Тип уязвимости (XSS, RCE, DoS, etc.)
   - Затронутые компоненты
   - Версии, которые подвержены уязвимости

2. **Шаги для воспроизведения**
   - Подробная инструкция по воспроизведению
   - Необходимая конфигурация
   - Скриншоты или логи (если уместно)

3. **Доказательство концепции (PoC)**
   - Код или скрипт для демонстрации
   - Тестовые данные

4. **Возможные последствия**
   - Что может сделать злоумышленник
   - Оценка критичности (низкая/средняя/высокая/критическая)

5. **Рекомендации по исправлению** (если есть)
   - Ваши предложения по фиксу
   - Ссылки на похожие уязвимости

## Процесс обработки

Мы следуем процессу ответственного раскрытия информации:

```
1. Отчёт получен → Подтверждение в течение 48 часов
2. Анализ уязвимости → Оценка в течение 5 дней
3. Разработка фикса → Зависит от сложности
4. Тестирование → Проверка исправления
5. Публикация advisory → Уведомление пользователей
6. Релиз патча → Новая версия с исправлением
```

### Временные рамки

- **Подтверждение**: 48 часов
- **Оценка критичности**: 5 дней
- **Разработка патча**: 7-30 дней (зависит от сложности)
- **Публикация**: После выпуска патча

## Типы уязвимостей

### Критические
- Удалённое выполнение кода (RCE)
- Обход аутентификации
- Утечка чувствительных данных (токены, ключи)
- DoS с возможностью удалённой эксплуатации

### Высокие
- XSS в Web UI
- CSRF атаки
- Несанкционированный доступ к API
- Утечка информации о сети

### Средние
- Частичный обход MAC фильтра
- Недостатки rate limiting
- Информация в логах

### Низкие
- Best practice нарушения
- Рекомендации по улучшению
- Documentation issues

## Безопасность конфигурации

### Рекомендации по безопасному развёртыванию

1. **Запуск от минимальных привилегий**
   - Используйте отдельную учётную запись
   - Ограничьте права доступа к файлам

2. **Защита конфигурации**
   ```json
   {
     "telegram": {
       "token": "используйте_переменные_окружения",
       "chat_id": "${TELEGRAM_CHAT_ID}"
     }
   }
   ```

3. **Сетевая безопасность**
   - Ограничьте доступ к API (порт 8080/8085)
   - Используйте firewall для защиты Web UI
   - Не открывайте порты наружу без необходимости

4. **Обновления**
   - Регулярно обновляйтесь до последней версии
   - Подпишитесь на уведомления о релизах
   - Проверяйте CHANGELOG на наличие security fixes

5. **Мониторинг**
   - Включите логирование всех событий
   - Настройте алерты на подозрительную активность
   - Регулярно проверяйте логи

## Security Features

### Встроенные функции безопасности

- **MAC фильтрация** - блокировка/разрешение устройств
- **Изоляция процессов** - запуск в отдельном процессе
- **Graceful shutdown** - корректная остановка без утечек
- **Rate limiting** - защита от flood атак (для логов)
- **Валидация конфигурации** - проверка перед запуском

### Проверка зависимостей

Мы регулярно проверяем зависимости на уязвимости:

```bash
# Проверка зависимостей
go list -m -u all
go list -m -json all | grep -i vuln

# Обновление зависимостей
go get -u ./...
go mod tidy
```
