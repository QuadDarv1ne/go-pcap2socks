## Базовая настройка

### Шаг 1: Установка

```bash
go install github.com/DaniilSokolyuk/go-pcap2socks@latest
```

### Шаг 2: Настройка

```bash
go-pcap2socks config
```

Отредактируйте файл конфигурации. Минимальный пример:

```json
{
  "pcap": {
    "network": "172.26.0.0/16",
    "localIP": "172.26.0.1"
  },
  "dns": {
    "servers": [
      { "address": "local" }
    ]
  },
  "routing": {
    "rules": [
      {
        "dstPort": "53",
        "outboundTag": "dns-out"
      }
    ]
  },
  "outbounds": [
    {
      "tag": "",
      "socks": {
        "address": "127.0.0.1:1080"
      }
    },
    {
      "tag": "dns-out",
      "dns": {}
    }
  ]
}
```

### Шаг 3: Запуск

```bash
sudo go-pcap2socks
```

### Шаг 4: Запишите настройки сети

При запуске вы увидите вывод:

```
Настройте ваше устройство со следующими параметрами сети:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  IP адрес:       172.26.0.2 - 172.26.255.254
  Маска подсети:  255.255.0.0
  Шлюз:           172.26.0.1
  MTU:            1486 (или меньше)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Шаг 5: Настройте устройство

На вашем устройстве:
1. Откройте **Настройки сети**
2. Установите **Статический IP** (любой из диапазона, например 172.26.0.100)
3. Установите **Маску подсети** (255.255.0.0)
4. Установите **Шлюз** (172.26.0.1)
5. Установите **DNS** (как шлюз или пользовательский)
6. Установите **MTU** если требуется

### Шаг 6: Тестирование

Подключитесь к интернету с вашего устройства и проверьте, что трафик маршрутизируется через прокси.

## Игровые консоли

### PlayStation 5 (PS5)

#### Настройка сети

1. **Настройки** → **Сеть** → **Настройки**
2. **Настроить подключение к интернету**
3. Выберите ваш тип подключения (Wi-Fi или LAN)
4. **Пользовательская настройка**
5. **Настройки IP-адреса**: Вручную
   - IP-адрес: `172.26.0.100` (или любой из диапазона)
   - Маска подсети: `255.255.0.0`
   - Шлюз по умолчанию: `172.26.0.1`
   - Основной DNS: `172.26.0.1`
   - Альтернативный DNS: `8.8.8.8`
6. **Настройки MTU**: Вручную
   - MTU: `1486` (или значение, показанное при запуске)
7. **Прокси-сервер**: Не использовать
8. **Тест подключения к интернету**

#### Примечания

- MTU должен быть установлен вручную
- Используйте точное значение MTU, показанное при запуске go-pcap2socks

### Nintendo Switch

#### Настройка сети

1. **Системные настройки** → **Интернет** → **Настройки интернета**
2. Выберите вашу сеть
3. **Изменить настройки**
4. **Настройки IPv4**: Вручную
   - IP-адрес: `172.26.0.100`
   - Маска подсети: `255.255.0.0`
   - Шлюз: `172.26.0.1`
   - Основной DNS: `172.26.0.1`
   - Вторичный DNS: `8.8.8.8`
5. **MTU**: Вручную
   - MTU: `1486` (или значение, показанное при запуске)
6. **Прокси-сервер**: Не использовать
7. **Сохранить** → **Тест подключения**

#### Примечания

- MTU критически важен для Switch
- Некоторые игры могут требовать проброса портов на вашем SOCKS прокси

### Xbox Series X/S

#### Настройка сети

1. **Настройки** → **Общие** → **Настройки сети**
2. **Дополнительные настройки**
3. **Настройки IP**: Вручную
   - IP-адрес: `172.26.0.100`
   - Маска подсети: `255.255.0.0`
   - Шлюз: `172.26.0.1`
   - DNS: `172.26.0.1`
4. **Альтернативный MAC-адрес**: Использовать по умолчанию
5. **Тест подключения к сети**

## Мобильные устройства

### iOS (iPhone/iPad)

#### Настройка Wi-Fi

1. **Настройки** → **Wi-Fi**
2. Нажмите ⓘ рядом с вашей сетью
3. **Настройка IP**: Вручную
   - IP-адрес: `172.26.0.100`
   - Маска подсети: `255.255.0.0`
   - Маршрутизатор: `172.26.0.1`
4. **Настройка DNS**: Вручную
   - DNS сервер: `172.26.0.1`
5. **Сохранить**

### Android

#### Настройка Wi-Fi (зависит от устройства)

1. **Настройки** → **Сеть и интернет** → **Wi-Fi**
2. Длительное нажатие на сеть → **Изменить сеть**
3. **Дополнительно**
4. **Настройки IP**: Статический
   - IP-адрес: `172.26.0.100`
   - Шлюз: `172.26.0.1`
   - Длина префикса сети: `16` (для /16)
   - DNS 1: `172.26.0.1`
   - DNS 2: `8.8.8.8`
5. **Сохранить**

## Продвинутые сценарии

### Сценарий 1: Маршрутизация только определённого трафика через прокси

```json
{
  "pcap": {
    "network": "172.26.0.0/16",
    "localIP": "172.26.0.1"
  },
  "dns": {
    "servers": [
      { "address": "local" }
    ]
  },
  "routing": {
    "rules": [
      {
        "dstPort": "53",
        "outboundTag": "dns-out"
      },
      {
        "dstPort": "80,443",
        "outboundTag": "proxy"
      }
    ]
  },
  "outbounds": [
    {
      "tag": "",
      "direct": {}
    },
    {
      "tag": "proxy",
      "socks": {
        "address": "127.0.0.1:1080"
      }
    },
    {
      "tag": "dns-out",
      "dns": {}
    }
  ]
}
```

Это маршрутизирует только HTTP/HTTPS через прокси, всё остальное напрямую.

### Сценарий 2: Блокировка определённых устройств

```json
{
  "routing": {
    "rules": [
      {
        "dstPort": "53",
        "outboundTag": "dns-out"
      },
      {
        "srcIP": ["172.26.0.100"],
        "outboundTag": "block"
      }
    ]
  },
  "outbounds": [
    {
      "tag": "",
      "direct": {}
    },
    {
      "tag": "block",
      "reject": {}
    },
    {
      "tag": "dns-out",
      "dns": {}
    }
  ]
}
```

Блокирует трафик с устройства с IP 172.26.0.100.

### Сценарий 3: Несколько прокси с балансировкой нагрузки

```json
{
  "routing": {
    "rules": [
      {
        "dstPort": "53",
        "outboundTag": "dns-out"
      },
      {
        "dstPort": "80",
        "outboundTag": "proxy1"
      },
      {
        "dstPort": "443",
        "outboundTag": "proxy2"
      }
    ]
  },
  "outbounds": [
    {
      "tag": "",
      "direct": {}
    },
    {
      "tag": "proxy1",
      "socks": {
        "address": "proxy1.example.com:1080"
      }
    },
    {
      "tag": "proxy2",
      "socks": {
        "address": "proxy2.example.com:1080",
        "username": "user",
        "password": "pass"
      }
    },
    {
      "tag": "dns-out",
      "dns": {}
    }
  ]
}
```

### Сценарий 4: Интеграция с VPN

```json
{
  "executeOnStart": [
    "sing-box run -c /etc/sing-box/config.json"
  ],
  "pcap": {
    "network": "172.26.0.0/16",
    "localIP": "172.26.0.1"
  },
  "dns": {
    "servers": [
      { "address": "local" }
    ]
  },
  "routing": {
    "rules": [
      {
        "dstPort": "53",
        "outboundTag": "dns-out"
      },
      {
        "dstIP": ["192.168.0.0/16", "10.0.0.0/8"],
        "outboundTag": "direct"
      }
    ]
  },
  "outbounds": [
    {
      "tag": "",
      "socks": {
        "address": "127.0.0.1:1080"
      }
    },
    {
      "tag": "direct",
      "direct": {}
    },
    {
      "tag": "dns-out",
      "dns": {}
    }
  ]
}
```

Запускает VPN перед go-pcap2socks, маршрутизирует локальную сеть напрямую.

## Устранение неполадок

### Устройство не может подключиться

**Проверьте:**
1. go-pcap2socks запущен
2. Настройки IP верны на устройстве
3. Брандмауэр разрешает подключения
4. IP шлюза совпадает с конфигом

**Решение:**
```bash
# Проверить, запущен ли
ps aux | grep go-pcap2socks

# Перезапустить
sudo go-pcap2socks
```

### Медленное соединение

**Возможные причины:**
1. Медленный SOCKS5 прокси
2. MTU слишком высокий
3. Перегруженность сети

**Решения:**
```json
// Уменьшить MTU
"pcap": {
  "mtu": 1400
}
```

- Попробуйте другой прокси-сервер
- Проверьте пропускную способность сети

### Ошибка разрешения DNS

**Проверьте:**
1. DNS-серверы доступны
2. DNS outbound настроен

**Решение:**
```json
"dns": {
  "servers": [
    { "address": "8.8.8.8" },
    { "address": "1.1.1.1" }
  ]
}
```

### Permission denied

**Решение:**
- **Windows**: Запуск от имени администратора
- **Linux/macOS**: Используйте `sudo`

```bash
sudo go-pcap2socks
```

### Режим отладки

Включите debug логирование:

```bash
SLOG_LEVEL=debug sudo go-pcap2socks
```

Включите захват пакетов:

```json
"capture": {
  "enabled": true,
  "outputFile": "debug.pcap"
}
```

Анализируйте с Wireshark:

```bash
wireshark debug.pcap
```

## Лучшие практики

1. **Используйте статические IP** для устройств для упрощения правил маршрутизации
2. **Тестируйте с минимальным конфигом** сначала, затем добавляйте сложность
3. **Мониторьте логи** для проблем: `SLOG_LEVEL=debug`
4. **Документируйте вашу настройку** для будущего использования
5. **Резервируйте конфиг** перед крупными изменениями
