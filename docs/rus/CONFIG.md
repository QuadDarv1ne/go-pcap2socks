# Руководство по настройке

Этот документ описывает все параметры конфигурации go-pcap2socks. Конфигурация использует формат JSON и хранится в файле `config.json`.

## Расположение файла конфигурации

- **По умолчанию**: В той же директории, что и исполняемый файл
- **Пользовательский**: Передать как аргумент: `go-pcap2socks /path/to/config.json`

## Открытие редактора конфигурации

```bash
go-pcap2socks config
```

Эта команда:
- Создаёт конфиг, если он не существует
- Открывает в редакторе по умолчанию:
  - **Windows**: Notepad
  - **macOS**: Текстовый редактор по умолчанию
  - **Linux**: Переменная окружения `$EDITOR` или `$VISUAL`, или nano/vim/vi

## Структура конфигурации

```json
{
  "executeOnStart": [],
  "pcap": { ... },
  "dns": { ... },
  "routing": { ... },
  "outbounds": [ ... ],
  "capture": { ... }
}
```

## executeOnStart

Команды для выполнения при запуске приложения. Полезно для запуска VPN-соединений или других предварительных требований.

### Синтаксис

```json
"executeOnStart": [
  "command1",
  "command2 arg1 arg2",
  "command3"
]
```

### Пример

```json
"executeOnStart": [
  "echo 'Запуск go-pcap2socks с VPN'",
  "sing-box run -c /etc/sing-box/config.json"
]
```

### Поля

| Поле | Тип | Обязательно | По умолчанию | Описание |
|------|-----|-------------|--------------|----------|
| Элементы | Массив строк | Нет | `[]` | Каждая строка — это shell команда |

### Примечания

- Команды выполняются по порядку
- Команды выполняются в фоновом режиме
- Ошибки логируются, но не останавливают запуск

## pcap

Управляет интерфейсом захвата пакетов и конфигурацией виртуальной сети.

### Синтаксис

```json
"pcap": {
  "interfaceGateway": "en0",
  "mtu": 1500,
  "network": "172.26.0.0/16",
  "localIP": "172.26.0.1",
  "localMAC": "aa:bb:cc:dd:ee:ff"
}
```

### Поля

| Поле | Тип | Обязательно | По умолчанию | Описание |
|------|-----|-------------|-------------|----------|
| interfaceGateway | Строка | Нет | Авто | Физический сетевой интерфейс для использования в качестве шлюза |
| mtu | Число | Нет | 1500 | Максимальный размер передаваемого блока (MTU) |
| network | Строка (CIDR) | **Да** | - | Диапазон адресов виртуальной сети |
| localIP | Строка (IP) | **Да** | - | Локальный IP-адрес в виртуальной сети |
| localMAC | Строка (MAC) | Нет | Авто | MAC-адрес виртуального интерфейса |

### Детали полей

#### interfaceGateway

- **Тип**: Строка
- **Формат**: Имя интерфейса (например, "en0", "eth0") или IP-адрес
- **По умолчанию**: Автоопределение шлюза по умолчанию
- **Пример**: `"en0"`, `"192.168.1.1"`

#### mtu

- **Тип**: Число
- **По умолчанию**: 1500
- **Рекомендуется**: 1486 или меньше (учитывает накладные расходы Ethernet)
- **Примечание**: Игровые консоли могут требовать определённых значений MTU

#### network

- **Тип**: Строка
- **Формат**: Обозначение CIDR
- **Пример**: `"172.26.0.0/16"`, `"192.168.100.0/24"`
- **Примечание**: Должен содержать localIP

#### localIP

- **Тип**: Строка
- **Формат**: IPv4 адрес
- **Пример**: `"172.26.0.1"`
- **Примечание**: Должен быть в диапазоне сети

#### localMAC

- **Тип**: Строка
- **Формат**: `aa:bb:cc:dd:ee:ff`
- **По умолчанию**: MAC физического интерфейса
- **Пример**: `"00:1a:2b:3c:4d:5e"`

### Пример конфигурации

```json
"pcap": {
  "interfaceGateway": "192.168.1.1",
  "mtu": 1486,
  "network": "172.26.0.0/16",
  "localIP": "172.26.0.1",
  "localMAC": "02:00:00:00:00:01"
}
```

## dns

Настраивает DNS-серверы для разрешения доменных имён.

### Синтаксис

```json
"dns": {
  "servers": [
    { "address": "local" },
    { "address": "8.8.8.8" },
    { "address": "1.1.1.1" }
  ]
}
```

### Поля

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| servers | Массив объектов | **Да** | Список DNS-серверов |

### Поля объекта Server

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| address | Строка | **Да** | Адрес DNS-сервера |

#### Значения address

- `"local"` - Использовать системный DNS
- IP-адрес - Конкретный DNS-сервер (например, `"8.8.8.8"`, `"1.1.1.1"`)

### Пример

```json
"dns": {
  "servers": [
    { "address": "local" },
    { "address": "8.8.8.8" },
    { "address": "208.67.222.222" }
  ]
}
```

## routing

Определяет правила маршрутизации трафика к различным исходящим соединениям.

### Синтаксис

```json
"routing": {
  "rules": [
    {
      "srcPort": "1024-65535",
      "dstPort": "80,443",
      "srcIP": ["192.168.1.0/24"],
      "dstIP": ["10.0.0.0/8"],
      "outboundTag": "proxy"
    }
  ]
}
```

### Поля

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| rules | Массив объектов | **Да** | Упорядоченный список правил маршрутизации |

### Поля объекта Rule

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| srcPort | Строка | Нет | Исходный порт(ы) для сопоставления |
| dstPort | Строка | Нет | Порт назначения для сопоставления |
| srcIP | Массив строк | Нет | Исходные IP-адреса/CIDR |
| dstIP | Массив строк | Нет | IP-адреса/CIDR назначения |
| outboundTag | Строка | **Да** | Тег исходящего соединения для использования |

### Формат портов

- **Один порт**: `"80"`
- **Несколько портов**: `"80,443,8080"`
- **Диапазон портов**: `"1000-2000"`
- **Комбинированный**: `"80,443,1000-2000,3000"`

### Формат IP

- **Один IP**: `["192.168.1.1"]` (автоматически /32)
- **CIDR**: `["192.168.0.0/24"]`
- **Несколько**: `["192.168.0.0/24", "10.0.0.0/8"]`

### Сопоставление правил

- Правила оцениваются по порядку
- **Первое совпадение побеждает**
- Если ни одно правило не совпало, используется исходящее соединение по умолчанию (пустой тег `""`)

### Примеры правил

```json
"routing": {
  "rules": [
    {
      "dstPort": "53",
      "outboundTag": "dns-out"
    },
    {
      "dstIP": ["192.168.0.0/16", "10.0.0.0/8"],
      "outboundTag": "direct"
    },
    {
      "dstPort": "80,443",
      "outboundTag": "proxy"
    },
    {
      "srcIP": ["172.26.0.100"],
      "outboundTag": "block"
    }
  ]
}
```

## outbounds

Определяет обработчики для исходящего трафика. Каждое исходящее соединение имеет тег для маршрутизации.

### Общие поля

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| tag | Строка | **Да** | Идентификатор для маршрутизации |

**Важно**: Пустой тег `""` обозначает **исходящее соединение по умолчанию** для несовпавшего трафика.

### Типы исходящих соединений

#### Direct

Маршрутизирует трафик напрямую в интернет без прокси.

```json
{
  "tag": "direct",
  "direct": {}
}
```

#### SOCKS

Маршрутизирует трафик через SOCKS4/SOCKS5 прокси.

```json
{
  "tag": "proxy",
  "socks": {
    "address": "proxy.example.com:1080",
    "username": "optional-user",
    "password": "optional-pass"
  }
}
```

**Поля:**

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| address | Строка | **Да** | Адрес и порт прокси-сервера |
| username | Строка | Нет | Имя пользователя для аутентификации SOCKS5 |
| password | Строка | Нет | Пароль для аутентификации SOCKS5 |

#### DNS

Специальный обработчик для DNS-запросов.

```json
{
  "tag": "dns-out",
  "dns": {}
}
```

#### Reject

Блокирует совпавший трафик.

```json
{
  "tag": "block",
  "reject": {}
}
```

### Примеры исходящих соединений

```json
"outbounds": [
  {
    "tag": "",
    "direct": {}
  },
  {
    "tag": "proxy",
    "socks": {
      "address": "127.0.0.1:1080",
      "username": "user",
      "password": "pass"
    }
  },
  {
    "tag": "dns-out",
    "dns": {}
  },
  {
    "tag": "block",
    "reject": {}
  }
]
```

## capture

Функция отладки для захвата пакетов в PCAP файл.

### Синтаксис

```json
"capture": {
  "enabled": true,
  "outputFile": "capture-debug.pcap"
}
```

### Поля

| Поле | Тип | Обязательно | По умолчанию | Описание |
|------|-----|-------------|--------------|----------|
| enabled | Boolean | Нет | `false` | Включить/выключить захват пакетов |
| outputFile | Строка | Нет | Авто | Путь для сохранения захваченных пакетов |

### Примечания

- Захватывает все Ethernet фреймы (включая MAC-адреса)
- Временная метка автоматически добавляется к outputFile, если не указана
- Полезно для устранения неполадок сети

## Переменные окружения

| Переменная | Значения | Описание |
|------------|----------|----------|
| `SLOG_LEVEL` | `debug`, `info`, `warn`, `error` | Установить уровень логирования |

### Пример

```bash
SLOG_LEVEL=debug sudo go-pcap2socks
```

## Проверка

После редактирования конфига проверьте синтаксис JSON:

```bash
# С помощью jq
jq . config.json

# С помощью python
python -m json.tool config.json
```

## Следующие шаги

1. Настройте ваши параметры
2. Выполните `go-pcap2socks config` для редактирования
3. Запустите `sudo go-pcap2socks`
4. Настройте сетевые параметры устройства как указано
5. Протестируйте подключение

Для примеров использования см. [Руководство по использованию](USAGE.md).
