# 🚀 Быстрый старт go-pcap2socks v3.19.19

## ✅ Новые возможности v3.19.19

- **Smart DHCP** — определение устройств по MAC, статические IP по типам
- **Device Detection** — 40+ производителей (Sony, Microsoft, Nintendo, Apple)
- **Engine Failover** — авто-переключение при ошибках движка
- **System Tuner** — авто-подбор буферов и таймаутов
- **Engine Auto-Select** — выбор лучшего движка (WinDivert/Npcap/Native)

## 📋 Требования

- ✅ Windows 10/11
- ✅ Права администратора
- ✅ Npcap установлен (https://npcap.com)
- ✅ Ethernet адаптер для раздачи интернета

## 🎯 Запуск за 3 шага

### Шаг 1: Автоконфигурация (первый раз)
```powershell
# PowerShell от имени администратора
cd M:\GitHub\go-pcap2socks
.\go-pcap2socks.exe auto-config
```

### Шаг 2: Запуск сервера
```powershell
# Используйте скрипт для автоматической проверки прав
.\run.bat

# Или напрямую от администратора:
.\go-pcap2socks.exe
```

### Шаг 3: Подключение устройства
Подключите PS4/телефон/компьютер по Ethernet и настройте:
- **IP адрес:** Автоматически (DHCP)
- **DNS:** Автоматически

Готово! Устройство получит IP из диапазона 192.168.137.10-250

    "network": "192.168.137.0/24",
    "localIP": "192.168.137.1",
    "mtu": 1486

  "dhcp": {
    "enabled": true,
    "poolStart": "192.168.137.10",
    "poolEnd": "192.168.137.250",
    "leaseDuration": 86400
  },
  "dns": {
    "servers": [
      {"address": "8.8.8.8:53"},
      {"address": "1.1.1.1:53"}
    ]
  },
  "routing": {
    "rules": [
      {"dstPort": "53", "outboundTag": "dns-out"},
      {"dstPort": "80,443,3478,3479,3480", "outboundTag": ""}

  },
  "outbounds": [
    {"tag": "", "direct": {}},
    {"tag": "dns-out", "dns": {}}
  ],
  "upnp": {
    "enabled": true,
    "autoForward": true,
    "gamePresets": {
      "ps4": [3478, 3479, 3480],
      "xbox": [3074, 3075],

## 🎮 Что читается из DHCP запроса устройства

| Option | Название | Пример | Описание |
|--------|----------|--------|----------|
| **12** | Host Name | "PS4-123" | Имя хоста устройства |
| **53** | Message Type | 1=Discover, 3=Request | Тип DHCP сообщения |
| **55** | Parameter Request | [1,3,6,15,28,...] | Список запрашиваемых параметров |
| **60** | Vendor Class | "MSFT 5.0" или "PSPC" | Идентификатор производителя |
| **61** | Client ID | Уникальный ID | Идентификатор клиента |

**Устройство само решает, какие параметры использовать** из тех, что отправляет сервер в DHCP OFFER/ACK.

## ❓ Решение проблем

### Сервис не запускается
1. Убедитесь, что запущен **от имени администратора**
2. Проверьте, что установлен **Npcap**: https://npcap.com
3. Проверьте, что **кабель Ethernet подключён**
4. Проверьте логи в `app.log`

### Устройство не видит сеть
1. Проверьте, что кабель подключён
2. Проверьте ARP таблицу: `arp -a | findstr 192.168.137`
3. Устройство должно быть в таблице ARP

### NAT тип 3 (строгий)
1. Включите UPnP на роутере
2. Или пробросьте порты вручную для IP устройства

### Ошибка "network adapter disconnected"
1. Проверьте, что кабель Ethernet подключён
2. Проверьте, что сетевой адаптер включён
3. Перезапустите сервис
