# Новые функции go-pcap2socks

## 📋 Обзор изменений

### Версия 2.0 - Полная автоматизация

#### 1. Обработка отключения адаптера
Программа теперь корректно обрабатывает отключение сетевого адаптера без спама ошибками:
- При отключении PS4 или кабеля Ethernet программа не падает
- Ошибки записываются в лог с уровнем Warning
- При восстановлении подключения работа продолжается автоматически

#### 2. Автоматический fallback на прямой интернет
Если SOCKS5 прокси (AmneziaVPN) становится недоступен:
- Трафик автоматически переключается на прямое подключение
- В логах появляется уведомление о переключении
- При восстановлении SOCKS5 трафик возвращается обратно
- Счётчик fallback-подключений доступен для мониторинга

#### 3. Система уведомлений
Windows Toast уведомления + логирование:
- **Info** — информационные сообщения
- **Success** — успешные операции
- **Warning** — предупреждения (например, fallback активирован)
- **Error** — критические ошибки

Уведомления дублируются в лог и показываются в трее.

#### 4. Работа в системном трее
Запуск в режиме трее с иконкой и меню:

```powershell
.\go-pcap2socks.exe tray
```

**Возможности:**
- Отображение статуса (Запущено/Остановлено)
- Быстрый доступ к конфигурации
- Авто-конфигурация
- Просмотр логов
- Управление сервисом

#### 5. Windows сервис
Установка и запуск как сервис Windows:

```powershell
# Установка сервиса
.\go-pcap2socks.exe install-service

# Запуск сервиса
.\go-pcap2socks.exe start-service

# Проверка статуса
.\go-pcap2socks.exe service-status

# Остановка сервиса
.\go-pcap2socks.exe stop-service

# Удаление сервиса
.\go-pcap2socks.exe uninstall-service
```

**Преимущества сервиса:**
- Автоматический запуск при загрузке Windows
- Работа в фоновом режиме
- Интеграция с диспетчером задач Windows
- Логирование в журнал событий Windows

### Версия 3.0 - Веб-интерфейс и автоматизация

#### 6. Веб-интерфейс
Полноценная панель управления через браузер:

```powershell
# Запуск веб-сервера (порт 8080)
.\go-pcap2socks.exe web

# Или только API (порт 8081)
.\go-pcap2socks.exe api
```

**Возможности:**
- 📊 Статистика трафика в реальном времени
- 🎮 Список подключенных устройств
- ⚙️ Управление сервисом (старт/стоп)
- 📈 Графики трафика
- 📝 Логи в реальном времени
- 📥 Экспорт статистики в CSV

**Доступ:** http://localhost:8080

#### 7. REST API
Программный доступ ко всем функциям:

```bash
# Получить статус
curl http://localhost:8081/api/status

# Запустить сервис
curl -X POST http://localhost:8081/api/start

# Остановить сервис
curl -X POST http://localhost:8081/api/stop

# Получить статистику трафика
curl http://localhost:8081/api/traffic

# Экспорт трафика в CSV
curl http://localhost:8081/api/traffic/export -o traffic.csv

# Получить список устройств
curl http://localhost:8081/api/devices

# Получить конфигурацию
curl http://localhost:8081/api/config

# Обновить конфигурацию
curl -X POST http://localhost:8081/api/config/update \
  -H "Content-Type: application/json" \
  -d '{"pcap":{"mtu":1472}}'

# Получить список профилей
curl http://localhost:8081/api/profiles

# Переключить профиль
curl -X POST http://localhost:8081/api/profiles/switch \
  -H "Content-Type: application/json" \
  -d '{"profile":"gaming"}'

# UPnP обнаружение
curl -X POST http://localhost:8081/api/upnp/discover
```

#### 8. Статистика трафика
Подробный учёт трафика по устройствам:

**Функции:**
- Учёт по каждому устройству (IP/MAC)
- Разделение на upload/download
- Подсчёт пакетов
- Отслеживание сессий
- Экспорт в CSV для анализа

**Пример CSV:**
```csv
Timestamp,IP,MAC,Hostname,Total Bytes,Upload Bytes,Download Bytes,Packets,Connected
2026-03-22T19:00:00+03:00,192.168.137.100,78:c8:81:4e:55:15,PS4,1048576,524288,524288,1000,true
```

#### 9. Профили конфигураций
Быстрое переключение между настройками:

```powershell
# Создать профиль
.\go-pcap2socks.exe config
# Сохранить как "gaming.json" в папке profiles

# Переключить профиль через API
curl -X POST http://localhost:8081/api/profiles/switch \
  -H "Content-Type: application/json" \
  -d '{"profile":"gaming"}'
```

**Предустановленные профили:**
- `default.json` — стандартная конфигурация
- `gaming.json` — оптимизировано для игр (низкий MTU)
- `streaming.json` — оптимизировано для стриминга (высокий MTU)

#### 10. UPnP интеграция
Автоматическая настройка маршрутизатора:

```powershell
# Обнаружить UPnP устройства
.\go-pcap2socks.exe upnp-discover

# Через API
curl -X POST http://localhost:8081/api/upnp/discover
```

**Возможности:**
- Автоматическое обнаружение роутеров
- Получение внешнего IP
- Проброс портов (для будущих версий)
- Настройка QoS

#### 11. Горячие клавиши
Быстрое управление без открытия окон:

**Комбинации:**
- `Ctrl+Alt+P` — Вкл/Выкл прокси
- `Ctrl+Alt+R` — Перезапуск сервиса
- `Ctrl+Alt+S` — Остановка сервиса
- `Ctrl+Alt+L` — Показать/скрыть логи

**Отключить горячие клавиши:**
```json
// В config.json
{
  "hotkey": {
    "enabled": false
  }
}
```

## 🔧 Конфигурация

Файл `config.json` поддерживает новые опции:

```json
{
  "pcap": {
    "interfaceGateway": "192.168.137.1",
    "network": "192.168.137.0/24",
    "localIP": "192.168.137.1",
    "mtu": 1472
  },
  "dns": {
    "servers": [
      {"address": "8.8.8.8:53"},
      {"address": "1.1.1.1:53"}
    ]
  },
  "routing": {
    "rules": [
      {"dstPort": "53", "outboundTag": "dns-out"}
    ]
  },
  "outbounds": [
    {
      "socks": {
        "address": "127.0.0.1:10808"
      }
    },
    {
      "tag": "dns-out",
      "dns": {}
    }
  ],
  "language": "ru"
}
```

**Автоматическая конфигурация:**
```powershell
.\go-pcap2socks.exe auto-config
```

## 📈 Мониторинг

### Проверка статуса fallback
В логах ищите сообщения:
```
SOCKS5 proxy unavailable, using direct connection
SOCKS5 proxy is available again
```

### Проверка сервиса Windows
```powershell
# Через приложение
.\go-pcap2socks.exe service-status

# Через Windows
sc query go-pcap2socks
```

### Логи
- Консольный вывод — в окне терминала
- Файл логов — `logs.txt` в папке приложения
- Журнал событий Windows — для сервиса

## 🐛 Решение проблем

### Сервис не запускается
1. Проверьте права администратора
2. Убедитесь, что Npcap установлен
3. Проверьте журнал событий Windows

### PS4 не подключается
1. Проверьте кабель Ethernet
2. Убедитесь, что ICS включен
3. Проверьте настройки сети на PS4

### SOCKS5 недоступен
1. Проверьте, работает ли AmneziaVPN
2. Убедитесь, что порт SOCKS5 правильный (по умолчанию 10808)
3. Трафик автоматически переключится на прямой режим

### Уведомления не показываются
1. Проверьте настройки уведомлений Windows
2. Разрешите уведомления для go-pcap2socks
3. Уведомления всё равно записываются в лог
